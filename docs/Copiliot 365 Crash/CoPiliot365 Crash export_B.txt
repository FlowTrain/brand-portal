How it works


Pattern test
AWKif ($0 ~ /^[ ]{2,}(@|cd|cp|...|\(|\{)/)Show more lines

^[ ]{2,} ‚Üí the line starts with two or more spaces (we avoid mangling non‚Äërecipe indents like define bodies that may have a single space).
(@|cd|cp|mv|mkdir|echo|...) ‚Üí after those spaces, the line appears to begin a shell command typical of a recipe (e.g., @, cd, git, echo, if, for, etc.).
This heuristic selects only likely recipe lines‚Äîwe don‚Äôt touch variable assignments or rule headers.



Substitution of leading spaces ‚Üí single TAB
AWKsub(/^[ ]+/, "\t", $0);Show more lines

sub() replaces the first run of leading spaces with one TAB (\t).
We leave the rest of the line intact.



Print every line
AWKprintShow more lines

Lines that didn‚Äôt match are printed unchanged.
Matches are printed with the new TAB.



Safe write‚Äëback
Shell> Makefile.fixed && mv Makefile.fixed MakefileShow more lines

We write to a temporary file first, then replace the original.




Why this fixes the error: make treats a TAB at the start of a line as the recipe separator. If your editor (or copy/paste) converted tabs to spaces, make can‚Äôt find the separator and halts with missing separator. Converting those leading spaces back to a real TAB fixes it.
